# cbx-cli Comprehensive Review — 2026-06-11

**Scope:** full repo at commit `0667951` (main). Read-only review — no code changed, nothing merged, no AWS/CB API calls made.
**Method:** 14 parallel read-only dimension reviewers + adversarial verification of every Medium+ finding (46 verifications: 19 confirmed, 22 severity-adjusted, 5 refuted) + a completeness critic, on top of the standard gates run locally. Severities below are **post-verification**.

---

## 1. Executive summary

**Overall health: good.** The codebase compiles clean, `go vet` is clean, the full test suite passes (including under `-race`), generated code has zero drift, `go.mod` is tidy, the four public facades are byte-for-byte pure re-exports, and there is **no shell-injection surface anywhere** (all LLM prompts go via stdin; no `sh -c`; scanner env is scrubbed of credentials). The audit pipeline's concurrency patterns are correct, and the retry/rate-limit companions honor `Retry-After`, `GetBody`, and context cancellation.

No Critical findings survived adversarial verification. The two findings initially rated Critical (legacy-tagged e2e tests "leaving invariants untested") were downgraded/refuted: the 100 MB guard *is* unit-tested tag-free (`internal/audit/parser_test.go:202,220`), and the legacy tag reflects a deliberate CLI-surface removal.

**Top 5 risks (ranked):**

1. **Secret values leave the machine via the grounded LLM prompt.** ECS container env-var *values* (`internal/audit/discover/aws/lister_ecs_native.go:315`) and Lambda env-var *values* (`describer_lambda.go:110`, `lister_lambda_native.go:179`) are serialized verbatim into the `claude -p` prompt by `writeResourceTable`/`serialiseInputs` (`internal/audit/llm_analyzer.go:713,759`). The Lambda path even computes a secret-key heuristic (`anyKeyLooksLikeSecret`) — and then ships the values anyway. *(High / Medium)*
2. **Library-contract erosion.** `core/` (documented as importable by other modules) imports `internal/` in five places (`core/api/v1/client.go:8-9`, `core/plan/runner.go:8`, `core/plan/llm_caller.go:13`, `core/plan/render.go:8`, `core/plan/llm_client.go:7`), and `pkg/audit` exposes an *invisible public surface*: aliased types (`Options`, `Result`, `AWSAuditContext`) carry exported fields of types that are **not** re-exported (`AccountPosture`, `LLMConnection`, `FindingProvider`, …), forcing downstream consumers into `internal/audit`. *(High ×2)*
3. **The published least-privilege IAM policy breaks discovery.** `docs/cbx-audit-aws-iam.json` is missing `backup:*` and `ecr:*` entirely, plus `s3:GetBucketLocation` — users who apply it get silent `permission_errors` for Backup vaults/plans, all ECR repositories, and skipped S3 buckets. *(High ×2, Medium)*
4. **CI-gate drift.** No lint job exists despite CONTRIBUTING.md:50 and README claiming one runs (94 outstanding golangci-lint issues, no `.golangci.yml`); `pkg/cmd.ExitCode` — the integer contract `main.go` and every embedder depends on — has no unit test; `directAssetURL` ↔ GoReleaser `name_template` coupling is untested. *(Medium ×3)*
5. **Reachable vulnerabilities.** `govulncheck` reports 8 reachable: `jose2go v1.5.0` (2 DoS CVEs, GO-2025-4123 / GO-2023-2409, reachable via the keyring file backend `internal/core/auth/keyring.go:104,113` — fixed in v1.7.0) and 6 stdlib findings fixed in go1.26.3/1.26.4 (toolchain bump). Adjacent: unbounded `io.ReadAll` on the knowledge-API response (`internal/audit/knowledge/client.go:148`). *(Medium)*

**Biggest strengths:** pure facades with doc-commented intent; zero-drift codegen with a CI gate; correct severity→exit-code mapping with no stray `os.Exit(1)` anywhere on audit paths; stdin-only prompt passing with `scrubEnv` credential stripping for scanner subprocesses (`internal/audit/adapter.go:72`); 100 MB stat-before-read guard with no decompression vector anywhere; keyring with no plaintext fallback; Sentry default-off (empty DSN in source builds) with key/path scrubbing; race-clean fan-out (`internal/audit/orchestrator.go:44`, `discover.go` feeder pattern).

**One discovery that reframes the brief:** the offline IaC CLI surface (`--pulumi-state` / `--terraform-state` / `--source` / `--scanners`) has been **removed from the CLI** — `pkg/cmd/audit.go:9-24` exposes only `cbx audit aws`. The offline guarantee now holds trivially at the CLI; it survives as a *library* property of `pkg/audit.Collect` (zero network iff `Options.MockScanners=true`). CLAUDE.md:52 and parts of the brief describe the pre-removal world.

### 3.2 Architecture & layering

Checked all imports across `cmd/`, `pkg/`, `core/`, `internal/`, `tools/`, `e2e/`; read all four facades in full. Facades are pure (invariant 1 PASS). The real defects are H2 (core→internal, 5 sites) and H3 (pkg/audit's invisible surface). `cmd/cbx/main.go` is 20 lines, not 10, because of the two internal imports (L1). The `internal/llm` vs `internal/core/llm` split is coherent but misnamed (L2). No import cycles. **Fix order: H3 first (additive re-exports, cheap), then H2 (requires a small `core`-local config type).**

### 3.3 Offline guarantee

The IaC CLI surface is removed — `cbx audit` exposes only `aws` (`pkg/cmd/audit.go`). For the *library* path: `Collect` with `MockScanners: true` is verified zero-network; the trap is M2 (zero-value `Options{}` → `AllScanners()` incl. prowler/trivy). The update check and telemetry are the only per-command network-capable code: update check is env- and arg-gated (`update.IsDisabled()`, `shouldRunUpdateCheck`), buffered-channel goroutine with a 3 s context, banner to stderr only; telemetry is empty-DSN in source builds and opt-in in release builds. **Verdict: guarantee holds; document the library-mode contract and fix M2's default.**

### 3.4 Security — exec & subprocess

No `sh -c` anywhere; all prompts via stdin (`llm_subprocess.go:46`, `llm_grounded.go:144`, `core/plan/cli_streamer.go:102`); argv built from typed slices; LLM output is parsed, never executed. `scrubEnv` strips `CB_API_KEY`/`ANTHROPIC_API_KEY`/`OPENAI_API_KEY` and (non-AWS scanners) all `AWS_*`. Gaps: M3 (no ctx on `runScanner`), L7 (double-exec version check), L8 (0o644 reports), L9 (unverified curl|tar suggestion), L10 (settable binary-name var). `upgradeDirect` is unimplemented — cbx never downloads-and-executes binaries itself today.

### 3.5 Security — credentials & data leakage
